# CLAUDE.md — WebYes Frontend

## Project overview

React + TypeScript frontend for WebYes, a web auditing platform. Consumes the FastAPI backend at `/api/`. Matches the Figma design (login page prototype: `https://www.figma.com/proto/EnJmxL2D0rd0wvWntxgKNp/`).

---

## Technology stack

| Concern | Choice |
|---|---|
| Framework | React 19 |
| Language | TypeScript (strict) |
| Build tool | Vite 8 |
| Routing | TanStack Router (file-based) |
| Data fetching | TanStack Query v5 |
| Styling | Tailwind CSS v4 (via `@tailwindcss/vite` plugin) |
| UI components | Radix UI primitives (shadcn-style, hand-rolled) |
| Charts | Recharts |
| State | Zustand (auth store only) |
| Toasts | react-hot-toast |
| Icons | lucide-react |
| HTTP | Native fetch (custom `client.ts` wrapper) |

## Routing conventions

TanStack Router generates `routeTree.gen.ts` automatically from `src/routes/`. Run `npm run dev` or `npm run build` to regenerate after adding/removing route files.

- `__root.tsx` — wraps everything, provides `<Toaster />`
- `_app.tsx` — auth guard + sidebar layout; all authenticated routes live under `_app/`
- Prefix `_` creates a pathless layout route (not reflected in the URL)
- `$param` in a filename becomes a URL param (`Route.useParams()`)
- Redirects use `throw redirect({ to: '/target' })` inside `beforeLoad`

## API proxy

The Vite dev server proxies `/api/*` → `http://localhost:8000` (configured in `vite.config.ts`). The backend runs on port 8000 by default.

---

## Auth flow

1. `POST /api/auth/login` → returns `{ access_token, token_type }`
2. Token stored in `localStorage` as `access_token`
3. Axios interceptor attaches `Authorization: Bearer <token>` to every request
4. On 401 response, token is cleared and the user is redirected to `/login`
5. Zustand `authStore` holds `{ user, token }` in memory; `localStorage` persists the token across page reloads

## Design system

**Primary color:** `#2563eb` (blue-600 in Tailwind)

**Score color scale:**
- ≥ 90 → green (`#22c55e`)
- 50–89 → amber (`#f59e0b`)
- < 50 → red (`#ef4444`)

**Auth pages:** Split screen — left: blue gradient panel (`#0038c8 → #1a5ff8 → #3b82f6`) + illustration; right: white form panel.

**App pages:** Fixed 224px sidebar + fluid main content area.

## Adding a new route

1. Create `src/routes/_app/<name>.tsx` for an authenticated page
2. Export `export const Route = createFileRoute('/_app/<name>')({ component: MyPage })`
3. Run `npm run dev` (or `npm run build`) — `routeTree.gen.ts` updates automatically
4. Add a sidebar link in `src/routes/_app.tsx` if needed

## Environment / proxy

No `.env` files needed for local dev — the Vite proxy handles the `/api` → backend forwarding.
For production, build the frontend and serve it behind a reverse proxy that also routes `/api` to the backend, or set `VITE_API_BASE` and update `client.ts` accordingly.
